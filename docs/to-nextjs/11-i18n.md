# 11 — i18n com next-intl

## Por que next-intl?

| i18next (atual)                        | next-intl                        |
| -------------------------------------- | -------------------------------- |
| Configuração manual no browser         | Integração nativa com App Router |
| Sem suporte a RSC                      | Funciona em Server Components    |
| Detecção de idioma via plugin separado | Detecção via middleware nativo   |
| Sem tipagem nativa                     | Tipagem via `createTranslator`   |

---

## Instalação

```bash
npm install next-intl
```

---

## Estrutura de arquivos

```
src/
├─ i18n/
│  ├─ request.ts        ← config por request (server-side locale)
│  ├─ routing.ts        ← defineRouting (locales suportados)
│  └─ locales/
│     ├─ pt-BR.json     ← copiar de src/shared/i18n/locales/pt-BR.json
│     ├─ en-US.json
│     └─ es-ES.json
└─ middleware.ts         ← detecção de idioma e redirecionamento
```

## `src/i18n/routing.ts`

```ts
import { defineRouting } from "next-intl/routing";

export const routing = defineRouting({
  locales: ["pt-BR", "en-US", "es-ES"],
  defaultLocale: "pt-BR",
  localeDetection: true,
});
```

---

## `src/i18n/request.ts`

```ts
import { getRequestConfig } from "next-intl/server";
import { routing } from "./routing";

export default getRequestConfig(async ({ requestLocale }) => {
  let locale = await requestLocale;

  if (
    !locale ||
    !routing.locales.includes(locale as "pt-BR" | "en-US" | "es-ES")
  ) {
    locale = routing.defaultLocale;
  }

  return {
    locale,
    messages: (await import(`./locales/${locale}.json`)).default,
  };
});
```

## Rotas com i18n

Com o routing do next-intl configurado, as rotas ficam:

```
/               → /pt-BR (redirect automático por locale)
/divine-books   → /pt-BR/divine-books
/dusk           → /pt-BR/dusk
/equipments     → /pt-BR/equipments
/stones         → /pt-BR/stones
```

Ou sem prefixo se usar `localePrefix: "as-needed"`:

```ts
export const routing = defineRouting({
  locales: ["pt-BR", "en-US", "es-ES"],
  defaultLocale: "pt-BR",
  localePrefix: "as-needed", // pt-BR fica sem prefixo na URL
});
```

## App Router layout com locale

### `src/app/[locale]/layout.tsx`

```tsx
import { NextIntlClientProvider } from "next-intl";
import { getMessages, getLocale } from "next-intl/server";
import { QueryProvider } from "@/providers/QueryProvider";
import { ThemeProvider } from "@/providers/ThemeProvider";
import type { ReactNode } from "react";

export default async function LocaleLayout({
  children,
}: {
  children: ReactNode;
}) {
  const locale = await getLocale();
  const messages = await getMessages();

  return (
    <html lang={locale} suppressHydrationWarning>
      <body>
        <NextIntlClientProvider messages={messages}>
          <QueryProvider>
            <ThemeProvider>{children}</ThemeProvider>
          </QueryProvider>
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

## Migração das chaves de tradução

As chaves atuais são compatíveis com next-intl. Apenas reorganizar para namespaces aninhados:

### Atual (`pt-BR.json`)

```json
{
  "divineBooks.filters.stat": "Atributo",
  "divineBooks.filters.status": "Situação"
}
```

### Novo formato next-intl (objeto aninhado)

```json
{
  "divineBooks": {
    "title": "Livros Divinos",
    "filters": {
      "stat": "Atributo",
      "status": "Situação"
    }
  }
}
```

## Troca de idioma no client

```tsx
"use client";

import { useRouter } from "next/navigation";
import { useLocale } from "next-intl";
import { routing } from "@/i18n/routing";

export function LanguageSwitcher() {
  const router = useRouter();
  const currentLocale = useLocale();

  const handleChange = (locale: string) => {
    // next-intl cuida do redirecionamento para a versão correta da rota
    router.replace(
      `/${locale}${window.location.pathname.replace(/^\/[^/]+/, "")}`,
    );
  };

  return (
    <select
      value={currentLocale}
      onChange={(e) => handleChange(e.target.value)}
      aria-label="Idioma"
    >
      {routing.locales.map((locale) => (
        <option key={locale} value={locale}>
          {locale === "pt-BR"
            ? "Português"
            : locale === "en-US"
              ? "English"
              : "Español"}
        </option>
      ))}
    </select>
  );
}
```

## Arquivos de locale

Copiar os arquivos atuais e reorganizar para formato aninhado:

```
src/i18n/locales/pt-BR.json  ← copiar de src/shared/i18n/locales/pt-BR.json
src/i18n/locales/en-US.json  ← copiar de src/shared/i18n/locales/en-US.json
src/i18n/locales/es-ES.json  ← copiar de src/shared/i18n/locales/es-ES.json
```

> Rodar um script de transformação para converter de flat keys para objeto aninhado:
> `"divineBooks.filters.stat"` → `{ "divineBooks": { "filters": { "stat": "..." } } }`
